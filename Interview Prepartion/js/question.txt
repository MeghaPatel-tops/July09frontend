9. What is the event loop in JavaScript?
The event loop is a mechanism in JavaScript that handles asynchronous operations and ensures that non-blocking code runs efficiently, even though JavaScript itself is single-threaded.

It continuously checks:

Is the call stack empty?

If yes, can we take a callback/task from the queue and run it?

This process keeps JavaScript responsive.
1. Call Stack

Where JavaScript executes your code line by line.

2. Web APIs / Browser APIs

Handle async tasks like:

setTimeout

fetch

DOM events

Promises

3. Task Queues

There are two main queues:

‚úî Callback (Macro) Task Queue

Contains tasks like:

setTimeout

setInterval

click events

‚úî Microtask Queue

Contains:

Promise .then()

async/await callbacks

MutationObserver

üëâ Microtasks have higher priority than macrotasks.

4. Event Loop

The event loop continuously:

Checks if the call stack is empty

If empty ‚Üí pushes tasks from the microtask queue first, then macrotasks

12. What is the difference between synchronous and asynchronous code?

Synchronous code runs one line at a time and blocks further execution until the current task finishes.

‚úî Executes in sequence
‚úî Each task waits for the previous one
‚úî Can cause blocking (slow operations freeze the program)

Asynchronous code allows tasks to run without blocking the main thread.

‚úî Does NOT wait for a task to finish
‚úî Frees the program to run other code
‚úî Uses callbacks, promises, async/await

Common async operations:

setTimeout

fetch

reading files

database calls

13. What is a Promise? Explain its states.
A Promise is an object that represents the eventual completion or failure of an asynchronous operation.

It allows you to write async code in a cleaner way compared to callbacks (avoids callback hell).

A Promise acts as a placeholder for a value that will be available now, later, or never.
| State         | Meaning                                               |
| ------------- | ----------------------------------------------------- |
| **pending**   | Initial state ‚Äî operation is still running            |
| **fulfilled** | Operation completed successfully (`resolve()` called) |
| **rejected**  | Operation failed (`reject()` called)                  |
============================================================================

14. What is async/await?
async/await is a modern JavaScript syntax (ES8) used to work with Promises in a cleaner and more readable way.

It allows you to write asynchronous code that looks like synchronous code.

When you put async before a function:

‚úî The function always returns a Promise
‚úî The returned value is automatically wrapped in a Promise

AWAIT pauses the execution of the function until the Promise is resolved or rejected.

‚úî Can only be used inside async functions
‚úî Makes asynchronous code behave like synchronous

16. What is event delegation?
Event Delegation is a JavaScript technique where instead of adding event listeners to multiple child elements, you add a single event listener to a parent element and handle events using event bubbling.

In simple words:
document.getElementById("parent").addEventListener("click", (e) => {
  if (e.target.tagName === "BUTTON") {
    console.log("Button clicked:", e.target.textContent);
  }
});
